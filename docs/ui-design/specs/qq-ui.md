# QQ 音乐大屏 UI 规格

QQ 音乐页面采用平板式结构：左侧导航、右侧内容区、底部常驻播放控制条。共享视觉和手柄规则见 `steam-deck-ui-rules.md`。

## 顶层页面

| 页面 | 目的 | 关键交互 |
| :--- | :--- | :--- |
| 推荐 | 默认入口，展示个性化推荐和新歌。 | `A` 播放或进入，`X` 更多操作，`L1/R1` 切页。 |
| 搜索 | 主动找歌入口。 | 搜索框 `A` 唤键盘，结果列表 `A` 播放，`X` 菜单，`L2/R2` 翻页。 |
| 我的音乐 | 登录态资产入口。 | 红心、自建歌单、收藏歌单；未登录时显示扫码入口。 |
| 智能电台 | “猜你喜欢 / 雷达推荐”沉浸播放。 | 无上一首；`Y` 红心，`X` 下一首，`Start` 播放 / 暂停。 |
| 正在播放 | 封面、歌曲信息、同步歌词。 | 歌词区上下手动滚动，`B` 恢复自动跟随。 |

## 推荐页

结构：

- 顶部两张大入口卡：猜你喜欢、雷达推荐。
- 中部推荐歌单轮播。
- 下方新歌或推荐歌曲网格。
- 底部播放条始终可通过 `View` 直达。

数据要求：

| 数据 | 用途 |
| :--- | :--- |
| 推荐歌单 | 首页轮播或网格。 |
| 新歌 / 推荐歌曲 | 首页可直接播放的歌曲列表。 |
| 歌曲封面缩略图 | 前端直接加载 CDN 缩略图，失败显示占位图。 |

## 搜索页

结构：

- 顶部搜索框和提交按钮。
- 分类 tab：单曲、歌单、专辑、歌手。
- 结果列表：歌名、歌手、专辑、时长、VIP / HQ 标识。

交互：

- 搜索框按 `A` 唤起软键盘。
- 单曲结果按 `A` 播放。
- 结果行按 `X` 打开上下文菜单。
- 长结果集用 `L2/R2` 翻页或分页加载。

## 我的音乐

未登录：

- 显示扫码登录入口。
- 不渲染空资产列表。

已登录：

- 顶部账户横幅：头像、昵称、会员标识。
- “我喜欢的音乐”置顶。
- 自建歌单和收藏歌单分区展示。

## 智能电台

“猜你喜欢”和“雷达推荐”共用同一个沉浸播放器，但入口和模式标识必须明确。

规则：

- 不提供上一首。
- 当前批次播完后由 bridge / provider 拉下一批。
- 页面内只保留三个主要操作：红心、播放 / 暂停、下一首。
- 底部常规播放条可隐藏，避免两个播放器同时出现。

## 正在播放

布局：

- 左侧三分之一：大封面、歌名、歌手。
- 右侧三分之二：同步歌词。

交互：

- 当前歌词行高亮。
- 十字键上下进入手动滚动。
- `B` 或超时恢复自动跟随。

## 上下文菜单

歌曲行和歌单卡片通过 `X` 打开菜单。菜单项按 P3 最小集实现：

1. 下一首播放。
2. 添加到队列。
3. 收藏到歌单。
4. 查看歌手或专辑。

菜单项不可用时要禁用或隐藏，不显示会失败的入口。

## P3 API 缺口

| 能力 | UI 用途 |
| :--- | :--- |
| 推荐歌单 | 推荐页轮播 / 网格。 |
| 新歌或推荐歌曲 | 推荐页歌曲网格。 |
| 搜索分类 | 搜索页单曲 / 歌单 / 专辑 / 歌手 tab。 |
| 搜索联想 | 输入前或输入中提示；可后置。 |
| 我喜欢的音乐 | 我的音乐核心入口。 |
| 自建 / 收藏歌单 | 我的音乐资产区。 |
| 歌词 | 正在播放页同步歌词。 |
| 收藏到歌单 | 上下文菜单操作。 |
| 歌手 / 专辑详情 | 上下文菜单跳转；可后置。 |

## 验收

- 未登录时，“我的音乐”不会渲染空数据或报错。
- 纯手柄能完成推荐播放、搜索播放、上下文菜单、播放条跃迁。
- 智能电台没有上一首入口。
- 后端返回缺字段、空列表、不可播歌曲时 UI 保持可恢复错误态。
