# Steam Deck 大屏 UI 规则

本文件是 `/music` 大屏路由的共享 UI 契约。QQ 与网易云页面可以有不同产品结构，但必须共用这里的视觉、焦点和宿主安全规则。

## 目标

- 纯手柄可用：无鼠标、无触屏也能完成找歌、播放、控制、退出。
- 像 Steam 游戏模式页面：不要照搬手机 App 的圆角卡片和密集排版。
- 不拖垮宿主：Steam UI 与插件前端在同一 Chromium 上下文，插件错误只能降级插件自身。

## 视觉规则

| 项 | 规则 |
| :--- | :--- |
| 背景 | 近黑背景，允许封面模糊光晕；不要大面积品牌色铺底。 |
| 卡片 | 封面图本身就是卡片；直角或 2-4px 小圆角；不要手机式大圆角容器。 |
| 焦点态 | 2-3px 细白描边，贴近元素边缘，允许轻微外发光和缩放。 |
| 列表选中 | 水平灰色渐变高亮条；必要时用 Steam 蓝左侧指示条。 |
| 品牌色 | Steam 蓝 `#1a9fff` 是主交互色；QQ 绿 / NCM 红只做 Logo、徽标、红心等点缀。 |
| 字体和层级 | 标题粗体，正文常规；小节标题可用全大写小字号灰色。 |
| 底部图例 | 使用底部实心深色图例条；按键提示必须实时跟当前焦点 / 页面模式语义同步，避免用户误按。 |

## 焦点模型

页面分两类焦点区域：

```text
主内容区：导航、搜索、列表、网格、歌词
播放控制区：底部播放条或 provider 专属播放器
```

硬规则：

- 所有可交互元素必须能被 `Focusable` 选中。
- 禁止 hover-only、touch-only、不可见点击热区。
- 网格和多列布局使用 `MAINTAIN_X`，避免上下移动时列错位。
- 搜索框聚焦后按 `A` 必须唤起 Steam 软键盘。
- 长列表必须分页、虚拟化或限制渲染量；不能一次性把大量节点塞进 DOM。
- 滑块聚焦后十字键左右用于微调；离开滑块必须有明确路径。

## 按键契约

| 按键 | 行为 |
| :--- | :--- |
| D-Pad / 左摇杆 | 焦点导航；在歌词区可手动滚动；在滑块上微调。 |
| A | 确认：播放、进入详情、提交搜索、刷新二维码。 |
| B | 返回或关闭；从播放控制区回到主内容区；取消歌词手动滚动。 |
| X | 上下文菜单；在 NCM 歌词页可切换歌词 / 热评。 |
| Y | 播放队列；在特殊模式下可作为红心或播放全部快捷键。 |
| View | 主内容区与播放控制区之间跃迁，并记住原焦点。 |
| Start | 全局播放 / 暂停；不依赖当前焦点。 |
| L1 / R1 | 顶层页面切换。 |
| L2 / R2 | 长列表翻页；当 L1/R1 已用于顶层页时，可用于二级 tab。 |
| Steam / QAM | 系统保留键，插件不绑定。 |

按键提示硬规则：

- 底部图例只展示当前可执行动作；不可用动作必须隐藏或置灰。
- 当 `X` / `Y` 在特殊模式下被覆盖（如红心、垃圾桶、歌词 / 热评切换）时，提示文案必须同步改成覆盖后的真实语义。
- 焦点移动、页面模式切换、队列空态、后端不可用都会改变按键能力，提示必须同帧更新，不能保留上一状态的旧提示。

## View 键跃迁

- 焦点在主内容区时按 `View`：记录当前焦点，跳到底部播放条的播放 / 暂停按钮。
- 焦点在播放控制区时按 `View` 或 `B`：回到之前记录的主内容区焦点。
- 如果原焦点已卸载，回退到当前页面第一个主要可交互元素。

## 宿主安全

这些规则继承 `docs/DESIGN.md` 的 UI 崩溃隔离要求：

- 所有 RPC、事件处理、计时器回调都要捕获错误，不把异常抛到全局。
- 后端死亡、断网、畸形数据只能渲染错误态，不能冻结 Steam UI。
- 图片用缩略图；失败显示占位图。
- 不 monkey-patch Steam 全局对象，不写全局命名空间。
- 组件卸载时清理监听器、计时器和路由。

## 验收

每个 provider 页面都必须能用实体按键完成：

1. 进入 `/music`。
2. 切换页面。
3. 搜索或打开推荐内容。
4. 选歌播放。
5. 暂停 / 继续。
6. 跳到播放控制区并返回。
7. 打开并关闭上下文菜单。
8. 在后端异常或断网时显示可恢复错误态。
