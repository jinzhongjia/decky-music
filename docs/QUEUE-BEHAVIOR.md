# Steam Deck 音乐插件：播放队列（Queue）行为设计规范

本规范定义 `decky-music` 在不同播放入口下的队列语义。队列真相源由 Python bridge 维护，前端 UI 只触发动作和展示状态。

## 1. 核心队列模式

队列分为两种互斥模式：**普通列表模式（Normal Queue）** 与 **电台流模式（Radio Stream）**。

### 1.1 普通列表模式

- **特征**：有限长度静态列表，如歌单、专辑、搜索结果。
- **控制**：用户可查看完整队列，可手动上一首 / 下一首。
- **循环策略**：支持单曲循环、列表循环、随机播放。
- **持久化**：随 `settings.json` 落盘，重启 Steam Deck 后可恢复。

普通队列持久化结构为 `"queue": {"items": [{id, media_mid, name, singer, cover, duration}], "index": 3}`。存 id 类字段（歌曲 id + QQ 解析 URL 所需的 media_mid）+ 展示字段（恢复后队列浮层 / 状态徽章直接显示真实信息），键白名单写入，**绝不存解析出的播放 URL（限时 vkey）**；播放时按 id 重新向 provider 解析。写入必须走临时文件 + `os.replace()` 原子替换。

### 1.2 电台流模式

- **特征**：动态、无限或分批拉取的歌曲流，如 NCM 私人 FM、QQ 猜你喜欢、QQ 雷达推荐。
- **无上一首**：电台只能往前走，不支持回退。
- **无播放模式切换**：单曲循环或随机在电台模式下无意义。
- **动态补水**：当当前批次播放到倒数第二首时，bridge 自动向 provider 请求下一批并追加到内存队列。
- **UI 表现**：电台沉浸页隐藏上一首和播放模式；队列浮层不展示未来曲目，只显示“正在收听电台”与退出入口。

## 2. 播放触发场景与队列策略

| 触发场景 | 用户操作 | 队列行为策略 | 模式 |
| :--- | :--- | :--- | :--- |
| 歌单 / 专辑 / 排行榜 | 播放全部 | 全量替换：清空当前队列，写入完整列表，从第 1 首播放。 | 普通列表 |
| 歌单 / 专辑 / 排行榜 | 点击列表中某首单曲 | 上下文替换：写入该单曲所在列表，索引定位到该单曲。 | 普通列表 |
| 搜索结果页 | 点击某首单曲 | 全量替换：写入当前页搜索结果，从选中曲目播放。 | 普通列表 |
| 任意列表单曲 | `X` -> 下一首播放 | 插队追加：插入到当前播放索引之后。 | 普通列表 |
| 任意列表单曲 | `X` -> 添加到队列末尾 | 尾部追加：追加到队列最后。 | 普通列表 |
| 智能电台 / 私人 FM | 点击电台入口 | 模式覆写：清空普通队列，切换电台流，拉第一批并开播。 | 电台流 |

### 2.1 模式切换互斥规则

- **普通列表 -> 电台流**：原普通列表被抛弃，进入电台流。
- **电台流 -> 普通列表**：电台流被丢弃，重新建立静态列表。
- **跨 provider 切换**：QQ 与 NCM Song ID 体系不通用，切 provider 时队列强制清空并停止播放。
- **登录态不随切换清除**：`settings.json` 中 `accounts` 按 provider 分 key 存储。切换只决定 spawn provider 时注入哪份 cookie。

## 3. Bridge 队列控制

bridge 必须处理 player 的 `ended` 事件以决定下一首。协议 v1 事件形态为 `{"ev":"player","type":"ended","data":{}}`，即 `protocol.ChildEvent(ev="player", type="ended")`。

自动切歌逻辑：

1. **单曲循环**：`seek(0)` 重新播放当前索引。
2. **普通列表**：索引 `+1`。到末尾时：
   - 列表循环：索引重置为 `0`。
   - 随机播放：从随机洗牌后的内部索引取下一首。
3. **电台流**：索引 `+1`。若当前已是内存批次倒数第二首，异步调用 provider 对应接口补水。

### 3.1 队列为空

- **触发**：初次安装、跨 provider 切换、用户清空队列。
- **表现**：Tab 状态徽章显示空态；正在播放页显示“暂无播放”；进度和切歌控件禁用。
- **拦截**：此时 `Start` 盲操无效，Footer Legend 不展示“播放 / 暂停”为可执行动作。

## 4. UI 队列浮层

普通大屏页面按 `Y` 打开队列浮层。QQ 智能电台、NCM 私人 FM 等专属沉浸页可按各自 UI 规格覆盖 `Y` 键，队列入口改由页面按钮或不展示未来队列。

普通模式下：

- 列表渲染当前队列所有歌曲。
- 当前播放歌曲行高亮，并显示播放态图标。
- 队列内 `X` 菜单支持移除该曲、清空队列。

电台模式下：

- 不展示未来曲目。
- 仅展示当前播放单曲卡片和“正在收听智能电台 / 私人 FM”文案。
- 提供“退出电台模式并清空”按钮。

## 5. 持久化总览

| 状态 | 是否持久化 | 说明 |
| :--- | :--- | :--- |
| 普通队列（ids + index） | 是 | 随 `settings.json` 落盘；只存 id，URL 重新解析。 |
| 电台流内容 | 否 | 动态推荐流存了无意义；只记 `queue_mode: "radio"` 与电台类型。 |
| 播放模式 / 音量 | 是 | bridge 统一持有。 |
| 登录态（双 provider cookie） | 是 | 按 provider 分 key，切换 provider 不清除，文件 `chmod 0600`。 |
| 跨 provider 切换后的队列 | 清空 | Song ID 体系互不通用，无保留意义。 |
